# SCHEMA: Affect As Resource Grade Over Gap Analysis

**Author**: codex
**Date**: 2026-03-28T03:51:47+1100
**Addresses**: affect as the resource-allocation projection of gap analysis in a second-order convergence loop
**Status**: Draft

## Summary

Affect can be modeled as the executive projection of gap analysis.

Its role is not only to say that a gap exists. Its role is to grade how many resources the system should recruit to close that gap.

In that sense:

affect = resource grade over gap state

This fits the second-order SDLC model:

- first-order loop tries to close the immediate gap
- second-order loop analyzes how the gap was experienced and what control response is needed
- affect is the compressed routing signal produced by that second-order analysis

## Schema

### Gap Inputs

The affect projection should be computed from signals such as:

- artifact delta
- evidence delta
- disambiguation delta
- uncertainty
- time pressure
- recurrence
- blast radius
- irreversibility
- available control

### Projection

A simple form is:

`affect_grade = severity + uncertainty + urgency + recurrence + stakes - control`

Where:

- `severity` means how large the current gap is
- `uncertainty` means how unclear the closure path is
- `urgency` means how soon the gap matters
- `recurrence` means whether the system is failing on this class of gap repeatedly
- `stakes` means how costly non-closure would be
- `control` means how much effective action is currently available

### Affect Buckets

Example buckets:

- `mild_concern`
  gap is real but closure path is visible
  local iteration is probably enough

- `worry`
  gap is growing, recurring, or less controllable than expected
  more attention and context should be recruited

- `stress`
  gap is materially threatening convergence
  lower-priority work should be suppressed and tighter control should activate

- `panic`
  control is collapsing under time and uncertainty pressure
  exploration should stop and containment or escalation should dominate

## Interpretation

Under this model, emotion is not noise on top of reasoning.

It is the grading function that tells the system how much of itself to recruit.

That includes:

- how much attention to allocate
- how much context to load
- how much planning depth to use
- whether to stay exploratory or become constrained
- whether to escalate to higher-order control
- whether to suspend lower-priority work

So a compact definition is:

emotion grades how much of the system should be recruited to close a gap

## Relation To Disambiguation

Disambiguation matters directly here.

If a gap is large because intent or context remains ambiguous, the affect projection should rise even when the artifact delta alone looks small.

That means affect is sensitive not only to missing output, but to unresolved interpretation burden.

This is why disambiguation memory changes the affect landscape:

- remembered disambiguations reduce uncertainty
- reduced uncertainty lowers affect grade for similar future gaps
- lower affect grade means fewer resources need to be recruited
- the system becomes cheaper to run on repeated edge classes

So disambiguation memory does not only improve convergence mechanically. It changes the emotional economics of the system.

## Working Thesis

Affect is the resource-allocation metric generated by second-order gap analysis. It compresses severity, uncertainty, urgency, recurrence, stakes, and available control into a routing signal for executive behavior. In a convergence system, this means emotion is the mechanism that grades how many resources should be recruited to close a gap. Disambiguation memory lowers that grade over time by shrinking uncertainty and tightening the operating surface for future iterations.
