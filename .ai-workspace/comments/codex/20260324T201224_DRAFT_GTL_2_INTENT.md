# GTL 2.x Intent

**Status**: Draft
**Date**: 2026-03-24
**Purpose**: State the purpose of GTL 2.x as a graph-first language while preserving its form as an embedded Python DSL/SDK.

---

## 1. Position

GTL 2.x is not a new standalone textual language.

It remains:

- an embedded Python DSL
- an SDK for defining workflow structure
- a programmatic language surface for graph-based system design

What changes in GTL 2.x is not the hosting language.

What changes is the clarity of the model:

- graph becomes the one first-class structural type
- operators become the typed effectful action surface
- graph functions become the reusable workflow/program abstraction
- composition, substitution, recursion, and higher-order graph operations become core semantics

---

## 2. Why GTL 2.x Exists

The current body of work proved useful ideas:

- typed topology
- evaluators
- recursive work
- event-sourced convergence
- partial composition and refinement

But it also carried unnecessary complexity:

- too much special terminology for structural roles
- composition as a special feature rather than a normal operation
- runtime language leaking into the structural model
- insufficiently explicit distinction between language and interpreter

GTL 2.x exists to simplify and stabilize the conceptual model.

---

## 3. Core Intent

GTL 2.x should let a designer express:

- graphs as workflow/program values
- typed graph interfaces
- operators over graph contracts
- reusable named workflows as graph functions
- lawful graph composition
- lawful graph substitution
- recursive graph application
- higher-order graph programming such as fan-out, fan-in, gating, and promotion

without needing a separate parser or standalone syntax.

The Python DSL/SDK remains the vehicle.

The semantics become cleaner.

---

## 4. Core Thesis

Everything structural is graph.

That means:

- subgraph is graph
- interface-bearing graph is graph
- bounded graph is graph
- reusable workflow graph is graph
- refined graph is graph
- composed graph is graph

This is the conceptual simplification GTL 2.x is meant to formalize.

---

## 5. What GTL 2.x Is For

GTL 2.x is for expressing mixed workflow programs that combine:

- deterministic operations
- probabilistic operations
- human/judgment operations

under one structural model.

The language should support:

- product design workflows
- implementation workflows
- discovery workflows
- review and approval flows
- self-hosting/self-description flows
- imported reusable workflow libraries

---

## 6. What GTL 2.x Is Not

GTL 2.x is not:

- a planner
- a business-priority engine
- a selection engine
- a standalone parser-first programming language
- a replacement for ABG runtime concerns such as event replay, retries, resets, and provenance

Those belong either above the language or in the interpreter/runtime.

---

## 7. Language and Runtime Split

GTL 2.x defines:

- what graphs are
- what operators are
- what graph functions are
- what composition means
- what substitution means
- what higher-order graph operations mean

ABG defines:

- how those declarations are interpreted
- how events are emitted
- how truth is replayed
- how convergence is computed
- how work lineage and run attempts are governed

This split is part of the intent, not just a later implementation detail.

---

## 8. Desired Outcome

At the end of GTL 2.x, the system should have:

- one clear structural ontology
- one clear reusable workflow abstraction
- one clean language/runtime boundary
- one path from product scenarios to graph programs to runtime interpretation

The result should be:

- simpler requirements
- cleaner ADRs
- clearer tests
- stronger product scenario coverage
- and a more stable base for future composition, libraries, and higher-order behavior

---

## 9. Guiding Statement

GTL 2.x is a graph-first Python DSL/SDK for expressing deterministic, probabilistic, and judgment-bearing workflow programs through graphs, operators, graph functions, and higher-order graph composition, with ABG serving as the event-sourced interpreter/runtime.
