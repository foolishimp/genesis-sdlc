## install.py Migration Logic — Detail

The `_migrate_provenance()` function (lines 564-733) bridges old → new:

1. **Scan** (lines 595-610): Looks for `fp_assessment` and `review_approved` events BY OLD NAME in the event stream. These string literals STAY — they describe what the function is looking for in pre-refactor logs.
2. **Re-emit** (lines 708-715): Currently re-emits as `fp_assessment` and `review_approved`. CHANGE to emit `assessed{kind: fp}` and `approved{kind: fh_review}` with appropriate payloads.
3. **Docstring** (lines 566-567): Update to describe the old→new transformation.

This is NOT a blind find-replace. The function's job is to know both vocabularies.
